## 1. 产品定位

FoodMap V1.0 是一个**本地优先的个人美食手账地图**。用户可以在地图上记录餐厅或美食地点，并为每个地点补充照片、评分、到访时间、文字笔记、标签和图层分类。

V1.0 不做大众榜单，不做社区，不做实时协作，也不追求复杂营销感。它首先是一个用户自己使用的地图记录工具：

> 把吃过、想再去、值得推荐、需要避雷的美食地点，像旅行手账一样钉在地图上。

### 1.1 一句话说明

**FoodMap：一张带照片、评分和回忆的私人美食地图。**

### 1.2 产品关键词

- 地图优先
- 本地记录
- 旅行手账
- 暖色纸感
- 低干扰操作
- 图层分类
- 照片缩略图
- 只读分享
- 导入导出

## 2. 方案四设计方向

方案四采用“旅行手账风”，但必须避免变成杂乱拼贴。设计重点不是堆贴纸，而是让地图工作台拥有温暖、可回忆、轻手账的气质。

### 2.1 视觉目标

| 维度 | 要求 |
| --- | --- |
| 整体感受 | 像一本干净的旅行美食手账，而不是复杂运营页面 |
| 地图表现 | 地图仍是主角，使用低饱和、暖色、浅纹理底图风格 |
| 卡片表现 | 纸张质感、圆角、细描边、轻阴影，不使用厚重玻璃拟态 |
| 装饰元素 | 可少量使用胶带、邮戳、手绘线条、纸角，但只作为氛围点缀 |
| 信息密度 | 比前一版更克制，用户一眼只看当前任务相关信息 |
| 移动端 | 单手可操作，底部抽屉为主，不使用复杂左右栏 |

### 2.2 设计克制规则

1. 每个主界面最多出现 1 个装饰性元素，如纸夹、邮戳、胶带角。
2. 地图上的图钉数量可以多，但卡片和工具按钮必须少。
3. 不在同一屏同时展开图层、筛选、详情、样式面板。
4. 所有编辑入口都隐藏在明确按钮后，不默认铺开表单。
5. 分享页只读，不能出现“编辑”“删除”“新增”等操作。
6. 移动端首屏只保留搜索、地图、定位、图层/筛选入口、底部抽屉。

## 3. V1.0 用户目标

### 3.1 核心用户

1. **美食记录型用户**  
   吃过一家店后，希望用地图、照片和文字记录体验。

2. **旅行规划型用户**  
   去一个城市前后，希望整理想吃清单、已吃店铺和城市美食记忆。

3. **朋友分享型用户**  
   希望把自己整理的一组美食地点以只读方式分享给朋友查看。

4. **本地收藏型用户**  
   想把常去店、想吃店、踩雷店分层管理，而不是散落在地图收藏夹或聊天记录里。

### 3.2 用户要完成的事情

- 我想在地图上新增一个美食地点。
- 我想给这个地点加照片、评分、到访时间和笔记。
- 我想用图层区分“必吃餐厅”“咖啡馆”“甜品饮品”“想去清单”。
- 我想按关键词、城市、评分、标签和到访时间找回记录。
- 我想隐藏某个图层，只看一类地点。
- 我想生成一份本地只读分享快照。
- 我想导出 `.foodmap.json`，在另一个浏览器或设备导入查看。

## 4. V1.0 范围

### 4.1 包含范围

| 模块 | V1.0 要求 |
| --- | --- |
| 个人工作台 | 用户在 `#/map` 中创建、查看、编辑、删除美食点位 |
| 本地分享页 | 用户在 `#/share/:snapshotId` 查看只读快照 |
| 本地持久化 | 数据刷新后不丢失 |
| 地图点位 | 支持地图点击创建、搜索结果创建、点位点击查看详情 |
| 图层管理 | 支持图层名称、颜色、图标、显隐、点位数量 |
| 搜索筛选 | 支持关键词、城市、标签、评分、到访时间 |
| 照片管理 | 支持上传照片、生成缩略图、详情展示 |
| 分享快照 | 支持从当前数据生成本地快照 |
| 导入导出 | 支持 `.foodmap.json` 导出与导入 |
| 响应式 | 桌面端和移动端都有可用布局 |

### 4.2 不包含范围

| 非目标 | 说明 |
| --- | --- |
| 不做后端 | V1.0 不依赖服务端 API |
| 不做账号登录 | 不做注册、登录、用户体系 |
| 不做多人协作 | 不支持共同编辑同一张地图 |
| 不做实时同步 | 不保证多设备自动同步 |
| 不做公网永久分享链接 | 分享只在本地快照和文件导入导出范围内成立 |
| 不做服务端照片存储 | 原图不上传服务器 |
| 不做公开榜单 | 不提供全网榜单、推荐算法、商家排名 |

## 6. 全局体验原则

### 6.1 地图第一

首屏就是地图，不做营销落地页。所有面板都浮在地图上，但不遮挡主要地图区域。用户进入后应立刻看到：

- 当前城市或默认地图区域
- 已记录的美食图钉
- 搜索入口
- 图层入口
- 新增地点入口

### 6.2 信息按需出现

V1.0 不在首页展示所有字段。默认首页只展示：

- 地图点位
- 图层数量
- 搜索
- 筛选入口
- 当前选中点位的摘要卡片

完整内容只在详情抽屉或详情页中出现。

### 6.3 编辑动作要轻

新增和编辑都必须能快速完成。必填字段最少化：

- 名称
- 经纬度
- 图层
- 评分
- 到访时间

地址、城市、标签、笔记、照片均可后补。

### 6.4 分享页必须可信

分享页要让用户明确知道：

- 这是本地只读快照。
- 不会修改原始记录。
- 不能公开永久访问。
- 如需跨设备查看，需要导入 `.foodmap.json`。

### 6.5 旅行手账风不等于低效率

视觉可以温暖，但交互必须清晰。按钮、输入框、状态反馈要像工具产品一样直接。

## 7. 页面 PRD：个人工作台 `#/map`

### 7.1 页面目标

让用户在一张全屏地图中完成美食点位的记录、整理、筛选和分享。

### 7.2 桌面端布局

推荐画布：1440 × 900。

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部搜索区：搜索地点 / 城市 / 筛选 / 导入导出 / 分享 / 新增     │
├───────────────┬─────────────────────────────┬────────────────┤
│ 左侧图层面板   │ 中央全屏地图                  │ 右侧详情抽屉     │
│ 260-300px      │ 图钉、比例尺、定位、缩放工具     │ 360-400px       │
│               │                             │                │
└───────────────┴─────────────────────────────┴────────────────┘
```

#### 7.2.1 顶部搜索区

| 元素 | 行为 |
| --- | --- |
| Logo + 产品名 | 显示 FoodMap 和短副标题“我的美食地图” |
| 搜索框 | 输入地点、菜系、商圈或店名 |
| 城市选择 | 默认“当前城市 / 全部城市”，可手动选择 |
| 筛选按钮 | 打开筛选面板 |
| 导入/导出 | 打开数据弹窗 |
| 分享 | 打开生成分享快照弹窗 |
| 新增地点 | 打开新增点位弹窗；若刚点击地图，则带入经纬度 |

#### 7.2.2 左侧图层面板

| 元素 | 行为 |
| --- | --- |
| 图层标题 | 显示“图层” |
| 新建图层按钮 | 打开图层创建表单 |
| 图层列表 | 显示图标、名称、点位数量、显隐状态 |
| 显隐按钮 | 点击后地图点位立即隐藏/显示 |
| 当前图层高亮 | 作为新增点位的默认图层 |
| 本地记录摘要 | 显示总点位数、图层数、最后更新日期 |

默认图层建议：

- 全部美食
- 必吃餐厅
- 咖啡馆
- 小吃快餐
- 甜品饮品
- 朋友聚餐
- 想去清单

#### 7.2.3 中央地图区

| 元素 | 行为 |
| --- | --- |
| 地图底图 | 暖色低饱和，保持道路和水系可读 |
| 图钉 | 图标来自图层，颜色来自图层颜色 |
| 点位悬浮摘要 | 桌面端 hover 或 click 显示小卡片 |
| 点位点击 | 打开右侧详情抽屉 |
| 地图点击 | 显示“在这里新增”轻提示，二次确认后进入新增表单 |
| 定位按钮 | 定位到当前附近或默认城市中心 |
| 缩放按钮 | 地图缩放 |
| 图层按钮 | 快捷打开图层面板或图层浮层 |

#### 7.2.4 右侧详情抽屉

未选中地点时，不强行显示大卡片。显示轻量空态：

```text
选择一个图钉，查看这家店的照片、评分和笔记。
```

选中地点后展示：

1. 照片头图或缩略图轮播
2. 店名
3. 评分
4. 图层标签
5. 自由标签
6. 到访时间
7. 地址 / 城市
8. 笔记
9. 照片缩略图列表
10. 编辑按钮
11. 删除按钮

详情抽屉宽度建议 360-400px。内容过长时抽屉内部滚动，不让整页滚动。

## 8. 页面 PRD：移动端 `#/map`

### 8.1 页面目标

让用户在手机上以单手操作完成“看地图、找点位、看详情、快速新增”。

推荐画布：390 × 844。

### 8.2 移动端布局

```text
┌──────────────────────────┐
│ 顶部搜索栏 + 筛选入口       │
├──────────────────────────┤
│ 全屏地图                  │
│ 图钉 / 定位 / 缩放          │
│                          │
├──────────────────────────┤
│ 底部操作栏 / 底部详情抽屉    │
└──────────────────────────┘
```

### 8.3 移动端交互

| 场景 | 行为 |
| --- | --- |
| 首次进入 | 展示全屏地图，底部为 64px 操作栏 |
| 点击图钉 | 底部抽屉展开到半屏，显示地点摘要 |
| 上滑详情 | 抽屉展开到接近全屏，展示完整详情 |
| 下滑详情 | 折叠回摘要状态 |
| 点击图层 | 打开底部图层抽屉 |
| 点击筛选 | 打开底部筛选抽屉 |
| 点击新增 | 打开新增表单页或全屏弹窗 |
| 点击分享 | 打开分享快照确认弹窗 |

### 8.4 移动端底部操作栏

默认显示 4 个入口：

- 图层
- 筛选
- 清单
- 新增

不建议把“导入/导出”放在移动端主操作栏。可以放到更多菜单里。

## 9. 页面 PRD：本地只读分享页 `#/share/:snapshotId`

### 9.1 页面目标

让查看者在同一应用内打开某个快照，浏览地图点位、图层和详情，但不能编辑。

### 9.2 分享页必须展示

| 元素 | 说明 |
| --- | --- |
| 分享标题 | 用户生成快照时填写，如“上海周末咖啡地图” |
| 快照时间 | 显示导出或生成时间 |
| 地图点位 | 展示快照中的点位 |
| 图层切换 | 可隐藏/显示图层，但不修改原数据 |
| 点位详情 | 展示照片缩略图、评分、到访时间、标签、地址、笔记 |
| 导入提示 | 如果快照不存在，提示导入 `.foodmap.json` |

### 9.3 分享页禁止展示

- 新增地点
- 编辑地点
- 删除地点
- 上传照片
- 修改图层
- 保存按钮
- 账号相关入口

### 9.4 分享页文案

```text
这是一个本地只读美食地图快照。
你可以查看图层和地点详情，但不会修改原始记录。
```

## 10. 核心功能需求

### 10.1 美食点位

| ID | 需求 | 优先级 | 验收标准 |

### 10.9 导入导出

| ID | 需求 | 优先级 | 验收标准 |
| --- | --- | --- | --- |
| IO-01 | 导出 `.foodmap.json` | P0 | 导出文件包含 places、layers、photos thumbnails、snapshot metadata |
| IO-02 | 导入 `.foodmap.json` | P0 | 校验通过后写入本地存储 |
| IO-03 | 导入失败反馈 | P0 | 结构错误时不污染现有数据，并展示错误提示 |

## 11. 用户流程

### 11.1 新增点位：地图点击

```text
进入 #/map
→ 点击地图空白处
→ 显示“在这里新增美食地点”轻提示
→ 点击确认
→ 打开新增地点表单
→ 填写名称、评分、到访时间、图层
→ 可选上传照片、添加标签和笔记
→ 保存
→ 地图出现新图钉
→ 详情抽屉打开
```

### 11.2 新增点位：搜索结果

```text
输入店名/商圈/菜系
→ 展示搜索结果列表
→ 点击结果
→ 地图定位到结果
→ 点击“保存为美食点位”
→ 打开新增表单并预填名称、地址、城市、经纬度
→ 保存
```

### 11.3 查看详情

```text
点击地图图钉
→ 地图聚焦该点
→ 桌面端右侧详情抽屉打开
→ 移动端底部详情抽屉半屏展开
→ 用户查看照片、评分、到访时间、地址、标签、笔记
```

### 11.4 编辑点位

```text
打开点位详情
→ 点击“编辑”
→ 打开编辑表单
→ 修改字段
→ 保存
→ 关闭表单
→ 详情和地图图钉同步更新
```

### 11.5 删除点位

```text
打开点位详情
→ 点击“删除”
→ 出现确认弹窗
→ 确认删除
→ 点位从地图和列表消失
→ 显示 toast：已删除地点
```

### 11.6 筛选地图

```text
点击筛选
→ 选择城市、标签、评分、到访时间
→ 点击“查看结果”
→ 地图只显示符合条件的点位
→ 顶部显示筛选条件数量
→ 点击“清空”恢复全部
```

### 11.7 生成分享快照

```text
点击“分享”
→ 输入快照标题
→ 确认生成
→ 系统生成本地快照
→ 显示分享页入口
→ 用户可打开只读页或导出 .foodmap.json
```

### 11.8 导出导入

```text
点击“导入/导出”
→ 选择导出当前地图或当前快照
→ 下载 .foodmap.json

clean profile 中打开应用
→ 点击导入
→ 选择 .foodmap.json
→ 校验通过
→ 生成本地分享快照
→ 打开只读分享页
```

## 12. 状态设计

```text
还没有美食图钉
```

说明：

```text
点击地图任意位置，或搜索一家店，开始记录你的第一份味觉记忆。
```

按钮：

#### 筛选无结果

```text
没有找到符合条件的地点。
试试放宽评分、标签或到访时间范围。
```

按钮：

```text
清空筛选
```

#### 分享快照不存在

```text
没有找到这个本地分享快照。
如果你是从其他设备打开，请先导入对应的 .foodmap.json 文件。
```

按钮：

```text
导入文件

```

### 12.2 加载状态

- 地图加载：显示纸感底色 + “正在展开地图...”
- 数据加载：显示小型 loading，不遮挡地图
- 照片处理：缩略图位置显示“正在生成缩略图”
- 导入处理：显示“正在校验文件”

### 12.3 错误状态

| 导入格式错误 | 这个文件不是有效的 FoodMap 数据包。 |
| 存储失败 | 保存失败，可能是浏览器存储空间不足。 |

### 13.1 色彩

| Token | 建议色值 | 用途 |
| --- | --- | --- |
| `--color-paper` | `#F6EBD6` | 页面纸张底色 |
| `--color-paper-light` | `#FFF8EA` | 卡片底色 |

| `--color-olive` | `#6F7F47` | 成功、自然、筛选确认 |
| `--color-gold` | `#D99A2B` | 星级评分 |
| `--color-danger` | `#B94A3A` | 删除、危险操作 |

### 13.2 字体

中文优先：

- 标题：PingFang SC Semibold / 思源黑体 Bold
- 正文：PingFang SC Regular / 思源黑体 Regular
- 数字：Inter / SF Pro

不建议把所有文字做成手写体。手写感只用于小标签、装饰标注或空状态插画，避免影响可读性。

### 13.3 卡片

- 圆角：16-22px
- 边框：1px 暖灰描边
- 阴影：轻阴影，避免厚重
- 背景：米白或浅牛皮纸色
- 内容：标题、信息、按钮之间留足呼吸感

### 13.4 图钉

| 图层类型 | 图标 | 建议色 |
| --- | --- | --- |
| 必吃餐厅 | star | 橙色 |
| 咖啡馆 | coffee | 咖啡棕 |
| 小吃快餐 | bowl | 橄榄绿 |
| 甜品饮品 | heart | 蜜桃粉 |
| 朋友聚餐 | pin / group | 蓝灰 |

## 16. UX 验收标准

### 16.1 桌面端

- 首屏可明确识别地图是主角。
- 左侧图层、顶部搜索、右侧详情不会互相遮挡。
- 点位详情打开后，地图仍可见。
- 新增、编辑、删除入口清晰。
- 筛选状态可见，清空筛选入口明确。
- 分享页不出现任何编辑按钮。

### 16.2 移动端

- 地图默认占据主要屏幕空间。
- 搜索栏不遮挡关键图钉。
- 底部抽屉可折叠，不永久遮住地图。
- 图层和筛选入口易触达。
- 新增流程不需要复杂跳转。
- 详情内容可滚动，按钮不重叠。

### 16.3 数据与安全感

- 刷新后数据不丢失。
- 删除前必须确认。
- 导入失败不能破坏当前数据。
- 分享页明确“本地只读”。
- 导出文件名清楚，例如：`foodmap-shanghai-cafe-2026.foodmap.json`。

## 17. V1.0 体验结论

FoodMap V1.0 采用方案四“旅行手账风”后，产品重点应从“功能很多的地图工具”转为“温暖、克制、可持续记录的个人美食地图”。

这一版的成败不取决于装饰是否足够像手账，而取决于三件事：

1. **地图是否始终是主角。**
2. **添加和查看一个美食点位是否足够顺。**
3. **本地分享和导入导出是否让用户觉得清楚、可信、安全。**

V1.0 可以先把图层、标签、评分、照片、到访时间、笔记和本地分享做扎实。复杂自定义字段、多人协作、云端同步、公开地图链接和推荐算法都应留到后续版本。
