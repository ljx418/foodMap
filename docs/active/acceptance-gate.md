# FoodMap P17 Acceptance Gate

## P17 必过门槛

| Gate | Pass Criteria | Blocker Conditions |
| --- | --- | --- |
| 文档一致 | PRD、目标架构、计划、里程碑、验收门槛、gap drawio、P17 contract 均指向 P17 体验可信度与移动主路径收束 | 文档仍只描述 P16 或互相矛盾 |
| 待确认入口 | 首页能看到待确认数量，且可进入待确认工作台 | 待确认地点只能在地图上散落寻找 |
| 待确认处理 | 每个待确认地点可确认候选、跳过或手动挪动；处理后状态同步 | 不可信坐标仍以普通已确认地点展示 |
| 详情页 IA | 状态、标签、核心操作、照片、评分/时间/地址、笔记、校准、导航有稳定顺序 | 详情页横向截断、不可滚动、操作不可发现 |
| 标签编辑 | 详情页可新增、删除、持久化自定义标签 | 标签只能显示不能编辑，或刷新后丢失 |
| 评分展示 | 百分制评分按 `(数字评分 - 65) / 35 * 5` 归一到五分制展示，并保留原始分 | 评分作为普通标签污染筛选或展示错误 |
| 手动挪动 | 详情页和移动端均可启动、取消、确认挪动图钉 | 移动端无法挪动或误触即保存 |
| 首页筛选栏 | 2048x768、1440x900、1280x820、768x900、390x844 下可见控件不越界、不半截显示 | 任一目标尺寸出现截断、遮挡或不可点击 |
| 图钉视觉 | 已核验、待确认、选中、扫街榜、参考层、高密度状态可区分；点击/缩放不闪烁 | 选中态不变、图钉闪烁、颜色含义混乱 |
| 移动端主路径 | 390x844 和 430x932 下可完成详情查看、标签编辑、挪动、地图跳转、分享入口 | 底部栏/抽屉/地图互相遮挡导致流程中断 |
| 外部地图安全 | 已确认地点可打开地图/导航；待校准地点只能展示安全 fallback | 不可信坐标提供误导性导航 |
| 分享图一致 | 分享图预览和导出使用当前筛选/视野，PNG 非空且地点数一致 | 空白导出或导出与当前地图明显不一致 |
| 真实数据回归 | 扫街榜 50 条、钉图易参考层、个人收藏、待确认队列共同参与验收 | 只用空数据或临时假数据通过验收 |
| Agent 边界 | Agent 可读待确认上下文，但不能绕过用户确认固化坐标 | Agent 直接改坐标、删除或保存未经确认地点 |
| 命令回归 | `npm run build`, `npm test`, `npx playwright test`, `npm run verify:scanlist` 全部通过 | 任一命令失败 |

## P17 Required Commands

```bash
npm run build
npm test
npx playwright test
npm run verify:scanlist
```

## P17 Required Fixtures

P17 验收必须包含以下真实或确定性样例，避免空数据通过：

| Fixture | Purpose |
| --- | --- |
| 扫街榜 50 条 | 验证推荐参考层和真实数据性能回归 |
| 钉图易参考层 100+ 条 | 验证参考层显隐、图钉视觉和性能 |
| 个人收藏 30+ 条 | 验证筛选、列表、详情、分享图和缩放性能 |
| 待确认个人地点 10+ 条 | 验证待确认工作台和风险处理 |
| 高风险坐标地点 3 条 | 验证水里/桥上/明显偏移的阻断提示 |
| 候选确认地点 3 条 | 验证候选选择固化坐标 |
| 手动挪动地点 2 条 | 验证取消、确认和持久化 |
| 长店名/长地址地点 2 条 | 验证详情页和筛选栏不截断 |
| 标签丰富地点 5 条 | 验证标签编辑、筛选和分享图摘要 |

## P17 Manual And Evidence Scenarios

1. 在 clean profile 打开 `#/map`，确认首页筛选栏控件完整。
2. 加载真实个人收藏和待确认队列，确认待确认数量可见。
3. 打开待确认工作台，处理一个候选确认、一个手动挪动、一个跳过。
4. 打开长店名详情页，在桌面和移动端检查无截断、可滚动。
5. 在详情页新增和删除标签，刷新后检查持久化。
6. 检查百分制评分归一到五分制展示，不作为普通标签显示。
7. 移动端启动手动挪动，取消一次、确认一次。
8. 缩放地图并点击多个图钉，检查选中态稳定且无闪烁。
9. 切换扫街榜、参考层、待确认、标签筛选，检查地图和列表计数一致。
10. 导出分享图，检查预览、PNG 非空、地点数与筛选结果一致。
11. 通过 Agent Bridge 读取待确认上下文，确认不能直接固化坐标。
12. 运行扫街榜基线，确认 50 条核验数据未被破坏。

## P17 Exit Conditions

P17 只有在以下条件全部满足时才能出门：

- 所有 P17 blocker gate 通过。
- 完整移动端用户历程可端到端验收。
- 待确认地点事实错误风险有明确用户确认、候选证据和手动纠错机制。
- 详情页、筛选栏、图钉、分享图在目标尺寸和真实数据下均有截图证据。
- Agent 边界、外部地图降级、无个人图钉导出等风险路径清楚。
- active 文档和 drawio 与实现一致。
- P17 final acceptance report 或阶段验收摘要已创建。

Historical P1-P16 gates and reports are archived or retained as completed baseline documents; this active gate describes P17 blocker conditions.
