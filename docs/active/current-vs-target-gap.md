## P17 Target State

P17 的目标不是增加后端或自动化替用户判断所有真实地点，而是在纯前端约束下让用户能放心纠错和传播：

1. 待确认地点集中可见，用户知道还有哪些地点不可信。
2. 每个不可信地点都有候选确认、跳过和手动挪动路径。
3. 详情页不再像表单堆叠，而是围绕查看、标签、纠错、导航和记录组织。
4. 移动端主路径可用，底部栏、抽屉、地图和详情不互相遮挡。
5. 图钉视觉在缩放和选中时稳定，不制造闪烁和误导。
6. 首页筛选栏多尺寸完整可见，不出现半截按钮。
7. 分享图导出反映当前筛选结果，适合直接传播。
8. 扫街榜、参考层、个人收藏和待确认队列共同参与验收。

## P17 Development Path

1. P17-1：文档冻结和 drawio 校验。
2. P17-2：实现待确认工作台和坐标可信度闭环。
3. P17-3：重构详情页信息架构、标签和评分展示。
4. P17-4：优化移动端查看、标签、挪动、地图跳转和分享入口。
5. P17-5：收束首页筛选栏和图钉视觉系统。
6. P17-6：升级分享图预览与导出一致性。
7. P17-7：执行真实数据性能、Agent 边界、总验收和 PRD 规格检视。

## P17 Risks And Controls

- 事实错误风险：待确认工作台必须把不可信坐标显性化，不能隐藏在普通详情中。
- 误操作风险：手动挪动必须支持取消和确认，不能拖动即永久保存。
- 移动端风险：所有主路径必须在 390x844 和 430x932 截图验收。
- 视觉误导风险：图钉颜色、选中态、榜单/参考层必须有稳定含义。
- 分享虚假风险：导出图必须读取当前筛选/视野状态，不能导出错集。
- 性能风险：验收必须加载真实数据组合，不能只用空地图。
- 文档漂移风险：PRD、架构、计划、门槛和 drawio 必须同时更新。
