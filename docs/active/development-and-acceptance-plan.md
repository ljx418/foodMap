# FoodMap P17 Development And Acceptance Plan

## P17 Summary

P17 的当前阶段目标是把 P16 已完成的真实地点联动能力收束为可日常使用、可纠错、可移动端完成的体验闭环。核心方向包括：待确认地点工作台、详情页信息架构重构、手动挪动图钉常驻入口、首页筛选栏收束、统一图钉视觉系统、分享图预览与导出、移动端主路径优化、真实数据量性能验收。

P17 不引入账号、后端、云同步或公网永久分享链接。P16 的真实地点候选、外部地图跳转、扫街榜 50 条基线和地图海报能力均作为回归基线保留。

## P17 Required Acceptance Scenarios

1. 清洁状态进入 `#/map`，扫街榜和参考图层作为显隐开关存在，不污染个人统计。
2. 首页筛选栏在 2048x768、1440x900、1280x820、768x900、390x844 下所有可见控件完整可点击。
3. 导入或 seed 真实个人收藏后，待确认数量可见，点击进入待确认工作台。
4. 待确认工作台展示原始店名、用户评分、风险原因、当前坐标、候选门店和处理动作。
5. 用户确认一个候选后，图钉位置、待确认状态、详情状态和列表计数同步更新。
6. 用户手动挪动一个地点，取消后坐标不变，确认后坐标持久化并记录校准状态。
7. 详情页可新增和删除标签，刷新后仍保留。
8. 详情页在窄桌面和移动端可滚动，地图不跟随详情滚动产生空白。
9. 图钉选中态颜色稳定变化，点击/缩放时不闪烁。
10. 移动端完成：打开待确认 -> 详情 -> 编辑标签 -> 挪动图钉 -> 打开地图 App fallback -> 返回。
11. 分享图预览和导出使用当前筛选结果，PNG 非空且地点数一致。
12. `npm run verify:scanlist` 仍证明 50 条扫街榜基线有效。

## P17 Audit Opinion

- 审计状态：`Plan Ready`。P17 目标已从“泛体验优化”拆成可开发、可验收、可审计的阶段目标。
- 编码前必须确认：PRD、目标架构、计划、里程碑、验收门槛、gap drawio 和 P17 contract 均描述同一范围。
- P17 的高风险点是：错点被隐藏、详情页再次截断、移动端主路径不可滚动、图钉状态误导、筛选与分享图不一致、性能验收只覆盖空数据。
- 若开发中发现真实 POI 校准需要外部付费/后端能力，应停在审计门槛，不得用不可验证假候选冒充已校准。

Historical P1-P16 plans and reports are archived or retained as completed baseline documents; this active plan describes the current P17 stage.
