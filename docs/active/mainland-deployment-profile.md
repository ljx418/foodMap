# FoodMap Mainland China Deployment Profile

## Decision

FoodMap should not use GitHub Pages as the primary production entry for mainland China users. GitHub Pages can remain an overseas/developer fallback, but the mainland user-facing route should be a domestic static host with a custom domain, ICP filing, HTTPS, and optional CDN acceleration.

Recommended production route:

```text
User in mainland China
  -> ICP-filed custom domain, for example foodmap.example.cn
  -> DNS CNAME to domestic CDN or static website endpoint
  -> Tencent Cloud COS or Alibaba Cloud OSS static website bucket
  -> dist/ generated by npm run build:mainland
  -> FoodMap static WebApp
  -> IndexedDB local data and .foodmap.json portability
```

## Why GitHub Pages Is Not Enough

- GitHub Pages is not a reliable mainland China access target.
- The accepted P25/P26 GitHub Pages URL remains valid as an overseas static deployment baseline, but it is not a mainland service guarantee.
- FoodMap is pure frontend and local-first, so it does not need a business backend for mainland deployment. It does need a mainland-reachable static host.

## Preferred Providers

| Route | Use When | Required External Setup | Notes |
| --- | --- | --- | --- |
| Tencent Cloud COS + CDN | You prefer Tencent Cloud console and domestic CDN integration | Tencent Cloud account, COS bucket, custom domain, ICP filing, HTTPS certificate | Good fit for a static WebApp; CDN domestic acceleration requires filed domain. |
| Alibaba Cloud OSS + CDN | You prefer Alibaba Cloud console and OSS tooling | Alibaba Cloud account, OSS bucket, custom domain, ICP filing, HTTPS certificate | Good fit for a static WebApp; OSS custom domain/CDN in mainland requires filed domain. |
| Domestic ECS/Nginx | You need full server control or custom access logs | Cloud VM, Nginx, domain, ICP filing, HTTPS certificate | More ops work; unnecessary unless static hosting is blocked. |
| Cloudflare/Netlify/Vercel/GitHub Pages | Overseas or development fallback only | External account; optional custom domain | Not reliable enough as mainland primary route. |

Gitee Pages is not a recommended route because the service has been suspended/unstable and cannot be treated as the production baseline.

## Build Profile

Use this command for mainland custom-domain root deployment:

```bash
npm run build:mainland
npm run verify:mainland:deployment
```

By default, the mainland build uses base path `/`, suitable for:

```text
https://foodmap.example.cn/#/map
```

If the app must be deployed under a subpath, set:

```bash
FOODMAP_MAINLAND_BASE_PATH=/foodmap/ npm run build:mainland
FOODMAP_MAINLAND_BASE_PATH=/foodmap/ npm run verify:mainland:deployment
```

Then the expected route is:

```text
https://foodmap.example.cn/foodmap/#/map
```

## Tencent Cloud COS + CDN Steps

1. Create a COS bucket in a mainland region close to target users.
2. Enable static website hosting.
3. Build FoodMap with `npm run build:mainland`.
4. Upload the full `dist/` directory to the bucket root.
5. Bind a custom domain, for example `foodmap.example.cn`.
6. Complete ICP filing for the domain before using mainland CDN acceleration.
7. Enable HTTPS with a valid certificate.
8. Configure CDN origin to the COS static website endpoint.
9. Configure cache policy:
   - `index.html`: short cache, preferably no-cache or minutes.
   - `sw.js`: short cache.
   - `assets/*`: long cache because file names are content-hashed.
10. Verify:

```bash
FOODMAP_MAINLAND_DEPLOY_URL=https://foodmap.example.cn/ npm run verify:mainland:deployment
```

## Alibaba Cloud OSS + CDN Steps

1. Create an OSS bucket in a mainland region close to target users.
2. Enable static website hosting with `index.html` as the default homepage.
3. Build FoodMap with `npm run build:mainland`.
4. Upload the full `dist/` directory to the bucket root.
5. Bind a custom domain, for example `foodmap.example.cn`.
6. Complete ICP filing for the domain before using mainland custom domain/CDN.
7. Enable HTTPS with a valid certificate.
8. Add CDN acceleration if public traffic or latency requires it.
9. Configure cache policy:
   - `index.html`: short cache.
   - `sw.js`: short cache.
   - `assets/*`: long cache.
10. Verify:

```bash
FOODMAP_MAINLAND_DEPLOY_URL=https://foodmap.example.cn/ npm run verify:mainland:deployment
```

## Acceptance Gate

The mainland deployment cannot be called accepted until:

1. The URL is not GitHub Pages and is reachable from mainland China networks.
2. The custom domain has ICP filing if hosted or accelerated inside mainland China.
3. HTTPS is enabled.
4. `npm run build:mainland` passes.
5. `npm run verify:mainland:deployment` passes locally.
6. `FOODMAP_MAINLAND_DEPLOY_URL=<mainland-url> npm run verify:mainland:deployment` passes remotely.
7. Browser smoke covers `#/map`, refresh, `#/share/:snapshotId` missing-share fallback, static assets, manifest, and service worker.
8. A Mate70 real-device screenshot or recording proves the mainland URL opens the app.

## Product Boundaries

The mainland deployment changes hosting only. It must not add or claim:

- account login;
- cloud sync;
- remote backup;
- multiplayer collaboration;
- permanent public share links;
- backend POI service;
- native HarmonyOS HAP/AppGallery delivery;
- offline map tiles;
- external realtime POI search completion.

User data remains local in IndexedDB. Cross-device portability remains `.foodmap.json`.
