| Workstream | Covered Milestones | Completion Definition |

| W18-A 坐标准确性与候选校准 | M18-2, M18-3, M18-4 | 10+ 待确认地点有候选确认、跳过或手动挪动闭环，刷新后状态一致 |

## P18 出门条件

- 用户能从待确认详情或工作台继续查找候选，且无 provider Key 时仍有 fallback。
- 候选展示来源、地址、置信度、坐标精度、证据和风险理由。
- 候选确认前不得修改地点坐标；候选确认后地图、列表、详情、筛选摘要和分享图事实一致。
- 手动挪动保存前必须展示新旧坐标、确认/取消和校准含义。
- 移动端详情首屏核心动作可见，长照片、长笔记、高级候选搜索按需展开。
- 首页筛选摘要能解释当前可见点数、待确认数、标签/状态/来源筛选和图层开关。
- 分享图支持标题编辑和当前视野/当前筛选结果模式，导出结果与选择一致。
- 500、1000、3000 点模拟数据下缩放、筛选、详情打开和分享预览可用。
- Agent 直接固化坐标、删除地点或隐藏待确认状态必须被拒绝。
- `npm run build`、`npm test`、`npx playwright test`、`npm run verify:scanlist` 全部通过。
- P18 final acceptance report 或阶段验收摘要必须创建后才能宣布出门。

Historical milestones are archived under `docs/history`; this active roadmap tracks P18.
