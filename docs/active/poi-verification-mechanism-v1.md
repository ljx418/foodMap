# POI 核对机制 V1

日期：2026-06-04

## 背景

扫街榜数据曾把榜单条目直接转成地图图钉，导致同名门店、分店地址和近似坐标混用。例如 `万松小院(荷花垄店)` 被放到错误区域。该问题不能靠逐条修正解决，必须增加数据准入机制。

## 核心规则

1. 榜单条目不是 POI。榜单只证明“这个名称出现在榜单”，不能直接证明地址和坐标。
2. 同一地点先语义去重，再核验坐标。去重时需要处理括号分店名、别名、总店/分店、商圈和道路名。
3. 至少两个独立来源一致，才允许作为确定图钉上图。
4. 单来源或地址冲突的条目只能作为候选，允许在推荐列表展示，但默认不上地图图钉。
5. 坐标精度必须单独标注；近似坐标不能标为精确坐标。

## 数据状态

| 状态 | 含义 | 能否上图 |
| --- | --- | --- |
| `unverified` | 缺少核验元数据或只有榜单名称 | 否 |
| `single_source` | 只有一个来源或同一平台内多个页面 | 否 |
| `multi_source_verified` | 两个以上独立来源在店名、分店、区县、地址或坐标上匹配 | 是 |
| `conflict` | 店名、分店、地址或坐标存在冲突 | 否 |

## 多信源标准

独立来源按 `independentSourceGroup` 分组：

- `amap`：高德榜单页、高德 POI 页。二者能增强同平台置信度，但不能单独满足“多信源”。
- `open-web`：搜索结果、商户官网、公开点评页面、新闻/攻略页面。
- `manual`：人工核验记录，例如用户明确指出位置错误并给出可核验区域。

通过门槛：

- 店名规范化后相似，且分店名/商圈不冲突。
- 至少两个独立来源的区县或地址片段一致。
- 如存在坐标，坐标间距离应小于 800 米；超过则进入 `conflict`。
- 置信度 `confidence >= 0.78`。
- `coordinateTrust` 不得为 `none` 或 `low`。

## 语义去重

当前实现位于：

- `src/recommendations/verification.ts`

处理策略：

- 去除括号内分店描述后生成基础名称，例如 `万松小院(荷花垄店)` 和 `万松小院（万松园店）` 的基础名都是 `万松小院`。
- 保留分店和地址片段生成 `canonicalKey`，用于区分同品牌不同门店。
- 使用 `duplicateGroupId` 合并同名/别名候选，避免同一店多次上图。

## 产品行为

- 右侧扫街榜仍展示候选条目，供用户查看榜单信息。
- 地图只显示 `verified` 或 `multi_source_verified` 的图钉；未核验候选不上图。
- 候选详情展示核验状态、置信度和风险说明。
- 未核验候选不能收藏为地图地点；避免用默认中心点制造新的错误图钉。

## 后续改进

- 接入高德开放平台 POI API，用 POI ID 拉取结构化地址和经纬度。
- 增加第二来源抓取器，例如公开搜索结果或商户公开页面。
- 增加批量核验报告，输出冲突项、同名重复项、低置信度项。
- 在 UI 中增加“只看已核验 / 查看候选”切换。
