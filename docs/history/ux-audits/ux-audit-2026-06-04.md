# FoodMap V1.0 UX Audit

Date: 2026-06-04

## Method

- Used UX Architect and UI Designer review criteria.
- Captured desktop screenshot at 1440x900 and mobile screenshot at 390x844.
- Ran automated browser smoke tests after fixes.

## Critical Findings

1. The map surface appeared blank when external map tiles failed to load.
   - Impact: users could not trust that they were adding pins on a real Wuhan map.
   - Fix: added Wuhan-centered Leaflet map with Gaode raster tiles and a local offline Wuhan fallback map showing Yangtze River, Han River, the three town areas, and key landmarks.

2. Map-click creation was too abrupt.
   - Impact: normal map exploration could unexpectedly open the full editor.
   - Fix: map click now opens a confirmation popover with coordinates; users explicitly choose "新增" before entering the editor.

3. The map-mode toast duplicated and overlapped with mobile controls.
   - Impact: noisy feedback and blocked bottom actions.
   - Fix: removed map-mode toast and replaced it with a quiet in-map status pill.

4. Empty detail state was not actionable.
   - Impact: users saw a passive empty panel instead of a next step.
   - Fix: right drawer now explains the expected action and provides a "新增地点" CTA.

5. Search placeholder implied external POI search.
   - Impact: users expected map provider search, while V1 currently searches local saved records.
   - Fix: placeholder changed to "搜索已保存地点、标签、城市".

6. Desktop side panels consumed map space by default.
   - Impact: the primary map experience felt cramped before users had any saved pins.
   - Fix: left layer panel and right detail panel are collapsed by default. Each side exposes only a half-round floating button at the map edge and can be expanded on demand.

7. Empty first screen had duplicated guidance.
   - Impact: the user saw both a map hint and a large empty-state card, reducing map focus.
   - Fix: removed the large empty overlay for the zero-record state and kept a compact map hint.

8. Header actions were too ambiguous on desktop and too crowded on mobile.
   - Impact: desktop icon-only controls were hard to understand; mobile text labels wrapped vertically.
   - Fix: desktop toolbar now shows short text labels. Mobile keeps compact icon buttons plus one wider "新增" button.

9. The create/edit form exposed too many fields at once.
   - Impact: first-time recording felt heavier than necessary.
   - Fix: coordinates are now placed in a collapsible "位置坐标" section, with name/layer/city/date shown first.

## Remaining UX Risks

- The offline Wuhan fallback is schematic. It preserves Wuhan geography and coordinate-based pinning, but it is not a full street-level map when remote tiles are unavailable.
- The right panel should evolve into a useful saved-place list when no pin is selected.
- The add/edit form is functional but dense; a staged form or compact required-first layout would reduce first-use friction.
- Header icon buttons rely on icon recognition. Tooltips or text labels could improve discoverability.

## Verification

- `npm run build`: passed.
- `npm test`: passed, 4 unit tests.
- `npx playwright test`: passed, 6 desktop/mobile browser tests.
